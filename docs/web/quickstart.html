Quickstart.

Save a checkpoint from a HuggingFace model, then launch the emulator against it. Two CLI calls. No additional config required.

1. Save a checkpoint

Morphling's morphling_cmd entrypoint exposes save for converting HuggingFace models into the emulator's checkpoint layout:

morphling_cmd save \
  --model "facebook/opt-125m" \
  --output <ckpt-path>

2. Launch the emulator

morphling_emulator --ckpt_path <ckpt-path>

This starts the proxy backend server, which loads the checkpoint and listens for device connections (default 0.0.0.0:39000). It runs until you stop it with Ctrl-C. Override the bind address with --listen_ip / --listen_port. Point one or more devices at it to drive emulated training (see Deployment).

3. Multi-device runs

For virtual fleets and physical-device deployments behind an Nginx stream proxy, jump to Deployment. A typical 4-device virtual run looks like:

docker run -dit --rm --name redis -p 6379:6379 redis
sleep 5

cd scripts
SPDLOG_LEVEL=debug python run_devices.py \
  --num_devices 4 \
  --model_name facebook/opt-125m \
  --backend proxy \
  --seq_length 128 \
  --batch_size 1 \
  --cfg ../config/proxy/svr.ini

4. Enable per-GEMM green contexts (optional)

To switch a CUDA SM partition per GEMM in forward and backward, install the linear hooks with a trace-driven controller:

from morphling.runtime.green_context import (
    GreenContextConfig, GreenContextController,
)
from morphling.hooks import apply_hooks

cfg = GreenContextConfig(
    enabled=True,
    trace_path="traces/second_level.trace",
    clock_mode="wall",
)
controller = GreenContextController.from_config(device_id=0, cfg=cfg)
apply_hooks("linear", greenctx=controller)

Full API and structured log format documented in Per-GEMM Green Contexts.
