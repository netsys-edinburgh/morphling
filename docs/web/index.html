Welcome to Morphling.

A measurement-driven emulator for distributed machine-learning training on heterogeneous edge devices. Run unmodified training scripts against a unified backend — real or emulated — at scales no physical fleet can reach.

Star on GitHub Read the paper (DOI)

Per-GEMM CUDA Green Contexts

Trace-driven SM partitioning routed through autograd hooks. Switch the active partition before every linear GEMM in forward and backward.

Pluggable Worker Pool

Round-robin or shortest-wait scheduling across XtGemmWorker (cuBLASXt) and CpuWorker (MKL) with identical public surface.

Zero-Copy Scatter-Gather

Inter-device transfers over libevent via evbuffer_add_reference with shared_ptr cleanup. No staging copies.

Pool-Based Memory

Pinned-host and aligned buffer pools, bucketed by power-of-2 size and mlock'd so latency stays flat under load.

Morphling powers what-if studies on workloads that would otherwise need a physical edge fleet: thousands of emulated devices on a single GPU server, with latency MAE on the order of milliseconds and temperature MAE within a few degrees Celsius of measured hardware.

Get Started

Morphling is shipped as a CUDA-enabled Docker image. The image is the source of truth — rebuild after any code change.

Installation

Build the Docker image, mount your GPUs, and verify the environment in under 5 minutes.

Quickstart

Save a checkpoint from a HuggingFace model and launch the emulator end-to-end with two CLI calls.
