Installation

The canonical Morphling environment is the Docker image defined by Dockerfile. The image is the source of truth — every Python or C++ change requires a rebuild.

Requirements

  • Linux host with Docker 20.10+ and Docker Compose 2.0+.
  • NVIDIA Container Toolkit (--gpus all) for GPU paths.
  • NVIDIA GPU with compute capability 7.0+. Development uses L40S.
  • Green-context partitioning needs driver 12.5 or newer.
  • At least 8 GB RAM and 10 GB free disk for the image and cache.

Build the image

Run from the repository root:

# enables BuildKit + ccache mount layer
DOCKER_BUILDKIT=1 docker build -t device-emulator:latest .

# or via the Makefile target
make docker-build
ccache is cached across rebuilds. The Dockerfile mounts type=cache,target=/ccache capped at 5 GB, so a second build only recompiles changed translation units. Inspect with docker run --rm device-emulator:latest ccache -s.

Start an interactive container

docker run --rm -itd --gpus=all \
  --ulimit memlock=-1:-1 \
  --cap-add IPC_LOCK \
  device-emulator bash

--cap-add IPC_LOCK and the unlimited memlock rlimit are required because Morphling's pinned-host and aligned buffer pools call mlock to stabilise latency.

Verify the install

Run the full Python test suite inside a freshly built image:

docker run --rm --gpus all device-emulator:latest \
  python3 -m pytest tests -v

The image builds all C++ test categories at image-build time (unit, CUDA/cuBLAS, XtGemm/worker, zerocopy, benchmarks). See the C++ test catalogue for the full list.

Native (non-Docker) development

Possible, but not the supported path. See docs/DEV_README.md for prerequisites and docs/troubleshooting.md for the common library-loading errors.
