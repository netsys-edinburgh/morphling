## 这个数字是否合理？

### ✅ 合理的原因

1. **Message 大小合理**
   - 1.13 MB ≈ 1 个矩阵块
   - Block size 通常是 128-1024
   - 对于 float32: 1024 × 1024 × 4 bytes ≈ 4 MB
   - 1.13 MB 对应约 290K floats (296×296) ✓

2. **序列化时间合理**
   - 1.13 MB 序列化 129 us
   - 吞吐量: 8.49 GB/s ✓ (合理范围)
   
3. **序列化占 99.7% 合理**
   - 正好符合你之前说的"占 1/3 ~ 1/2"
   - 实际上占了更多（因为反序列化极快）
